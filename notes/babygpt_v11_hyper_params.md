# BabyGPT v11: 超参数调优

## 概述

v11 在 v10 的基础上进行了**超参数调优**，模型架构保持不变，但通过增大模型规模、扩展上下文窗口、调整学习率等手段，显著提升了模型性能。这是 BabyGPT 演进路线中第一个"生产级"配置。

## v10 vs v11 核心差异对比

### 超参数对比

| 参数 | v10 | v11 | 变化 |
|------|-----|-----|------|
| `batch_size` | 32 | 64 | × 2 |
| `block_size` | 8 | 256 | × 32 |
| `learning_rate` | 1e-2 (0.01) | 3e-4 (0.0003) | ÷ 33 |
| `n_embed` | 32 | 384 | × 12 |
| `n_head` | 4 | 6 | × 1.5 |
| `n_layer` | 3 | 6 | × 2 |
| `dropout` | 0.2 | 0.2 | 不变 |

### 架构不变

v11 的模型架构与 v10 完全相同，所有改进都来自超参数调整。

## 各超参数的作用

### 1. n_embed：嵌入维度

```
v10: n_embed = 32   → 每个 token 用 32 维向量表示
v11: n_embed = 384  → 每个 token 用 384 维向量表示
```

**影响**：
- **表达能力**：384 维可以编码更多信息
- **参数量**：Embedding 层参数增加 12 倍
- **计算量**：线性层计算量增加约 144 倍（12²）

**类比**：
| n_embed | 类比 |
|---------|------|
| 32 维 | 用 32 个关键词描述一个概念 |
| 384 维 | 用 384 个关键词描述，更精确、更细致 |

### 2. n_layer：网络深度

```
v10: n_layer = 3  → 3 个 Transformer Block
v11: n_layer = 6  → 6 个 Transformer Block
```

**影响**：
- **抽象层次**：更深的网络可以学习更复杂的层次化特征
- **参数量**：线性增加
- **梯度传播**：需要残差连接和 LayerNorm 支持

**类比**：
| n_layer | 类比 |
|---------|------|
| 3 层 | 开 3 轮会议讨论问题 |
| 6 层 | 开 6 轮会议，每轮理解更深 |

### 3. n_head：注意力头数

```
v10: n_head = 4   → 4 个注意力头，每个 head_size = 8
v11: n_head = 6   → 6 个注意力头，每个 head_size = 64
```

**head_size 计算**：
```
head_size = n_embed // n_head
v10: 32 // 4 = 8
v11: 384 // 6 = 64
```

**影响**：
- **并行模式**：更多头可以同时捕捉更多不同的关联模式
- **每个头的容量**：head_size 从 8 增加到 64，每个头的表达能力更强

### 4. block_size：上下文窗口

```
v10: block_size = 8    → 只能看前 8 个 token
v11: block_size = 256  → 可以看前 256 个 token
```

**这是最大的变化（32 倍）**，影响深远：

| 方面 | v10 (block_size=8) | v11 (block_size=256) |
|------|-------------------|---------------------|
| 上下文长度 | 约 8 个汉字 | 约 256 个汉字 |
| 长距离依赖 | 无法学习 | 可以学习 |
| Position Embedding | 8 × 32 = 256 参数 | 256 × 384 = 98,304 参数 |
| 注意力矩阵 | (B, 8, 8) | (B, 256, 256) |

**类比**：
| block_size | 类比 |
|------------|------|
| 8 | 只能记住最近 2 秒的内容 |
| 256 | 可以记住最近 1 分钟的内容 |

### 5. batch_size：批次大小

```
v10: batch_size = 32
v11: batch_size = 64
```

**影响**：
- **梯度稳定性**：更大的 batch 提供更稳定的梯度估计
- **GPU 利用率**：更好地利用 GPU 并行能力
- **内存占用**：需要更多显存

### 6. learning_rate：学习率

```
v10: learning_rate = 1e-2  (0.01)
v11: learning_rate = 3e-4  (0.0003)
```

**为什么需要更小的学习率？**

| 原因 | 说明 |
|------|------|
| 模型更大 | 参数更多，优化空间更复杂 |
| 梯度更复杂 | 深层网络的梯度方差更大 |
| 防止震荡 | 大模型需要更谨慎的步长 |

**经验法则**：模型参数量增加，学习率通常需要相应减小。

## 超参数调优的原则

### 1. 平衡规模与资源

| 规模 | 优点 | 缺点 |
|------|------|------|
| 太小 | 训练快、资源少 | 表达能力弱 |
| 太大 | 表达能力强 | 训练慢、需要大显存 |

### 2. 关键比例关系

```
n_embed / n_head = head_size  # 每个 head 的维度
n_embed × 4 = FFN 中间层维度  # FFN 扩展倍数
```

这些比例在 GPT 系列中基本保持一致。

### 3. 学习率与模型规模

经验公式（来自 GPT-3 论文）：
```
learning_rate ≈ 0.003239 - 0.0001395 × log(parameters_in_millions)
```

更大的模型需要更小的学习率。

### 4. block_size 与任务匹配

| 任务 | 推荐 block_size |
|------|----------------|
| 短文本生成 | 128 ~ 256 |
| 长文档处理 | 512 ~ 2048 |
| 代码生成 | 1024 ~ 4096 |

## v11 配置的来源

v11 的超参数配置参考了 GPT-2 Small：

| 参数 | GPT-2 Small | BabyGPT v11 |
|------|-------------|-------------|

| block_size | 1024 | 256 |

v11 约为 GPT-2 Small 的 1/10 规模，适合学习和实验。

## 总结

| 版本 | 核心变化 |
|------|----------|
| v10 | Dropout，防止过拟合 |
| v11 | 超参数调优，模型规模扩大 |

v11 的超参数调整体现了 Transformer 扩展的核心原则：

- **更大更深**：更多层、更大维度 → 更强表达能力
- **更长上下文**：更大的 block_size → 捕捉长距离依赖
- **更小学习率**：大模型需要更谨慎的优化

**v11 是 BabyGPT 演进路线中第一个具备实用价值的配置**，可以生成连贯的宋词风格文本。

从 v1 到 v11，我们一步步构建了一个完整的 GPT 风格模型：

```
v1:  Embedding + Linear          (~2K 参数)
v2:  + Position Embedding
v3:  + Self-Attention
v4:  + Multi-Head Attention
v5:  + Feed-Forward Network
v6:  + Block 封装
v7:  + 残差连接
v8:  + 投影层 + FFN 扩展
v9:  + LayerNorm
v10: + Dropout
v11: 超参数调优                  (~12.8M 参数)
```

这一路走来，每一步都有明确的动机和原理支撑，这正是深入理解 LLM 的最佳路径。
